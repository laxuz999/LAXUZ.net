# Spec.md — カレンダー登録アプリ 機能仕様書

**バージョン**: 1.5  
**最終更新**: 2026-04-21  
**対象ファイル**: `calendar-app.html`, `api.php`, `config.php`

---

## 1. アプリケーション概要

### 1.1 目的

エアコン・家電クリーニング業者が受け取る作業依頼の通知テキストを解析し、
Google カレンダーへの予定登録を最小限の操作で完了させる。

### 1.2 対象ユーザー

LAXUZ / ユアマ系クリーニング業者のスタッフ（シングルユーザー、認証なし）。

### 1.3 利用フロー概要

```
①アプリを開く → ②予定文が自動ペースト（またはキーボード入力） 
→ ③「解析」ボタン → ④AI が構造化 → ⑤確認・編集画面 
→ ⑥「Google カレンダーで開く」 → ⑦カレンダーに予定が入る
```

## 2. 画面仕様

### 2.1 ページ 1 — 入力画面（`#page-input`）

#### 表示内容

| 要素 | 内容 |
|---|---|
| ヘッダーアイコン | 📅（グラデーション背景） |
| ヘッダータイトル | 「カレンダー登録」 |
| サブテキスト | 「コピーした予定文をカレンダーに登録」 |
| テキストエリア | `#inputText`（10行分の高さ） |
| 文字カウント | リアルタイムで「〇〇文字」表示 |
| 解析ボタン | 「✦ 解析して登録内容を作成」（グラデーション） |
| エラー表示エリア | `#errorArea`（エラー時のみ表示） |

#### 自動クリップボード貼り付け

- ページロード時に `navigator.clipboard.readText()` を実行
- クリップボードにテキストがある場合、`#inputText` に自動セット
- 失敗（権限拒否等）は静かに無視する（エラー表示しない）

#### バリデーション

- テキストエリアが空の場合: 「予定文を入力してください」をエラーボックスで表示
- 解析中はボタンを `disabled` にし「解析中…」テキストに変える

### 2.2 ページ 2 — 確認・編集画面（`#page-result`）

#### 表示内容

| フィールド | ID | 入力タイプ | 説明 |
|---|---|---|---|
| タイトル | `#f-title` | `input[type=text]` | AI 生成タイトル（編集可） |
| 開始日時（JST） | `#f-start-jst` | `input[type=text]` | `YYYY/MM/DD HH:mm` 形式 |
| 終了日時（JST） | `#f-end-jst` | `input[type=text]` | `YYYY/MM/DD HH:mm` 形式 |
| 場所 | `#f-location` | `input[type=text]` | 住所（マンション名等は除く） |
| 詳細 | `#f-detail` | `textarea` | 入力テキスト全文 |

#### ボタン

| ボタン | 色 | 動作 |
|---|---|---|
| 📅 Google カレンダーで開く | グリーン | `buildGcalUrl()` で URL を生成し `window.open()` で新タブ |
| 🔗 URL をコピー | グレー | 同 URL を `navigator.clipboard.writeText()` でコピー |
| ← 戻る | テキストリンク | ページ 1 に戻る（入力内容は保持） |

## 3. 機能仕様

### 3.1 テキスト解析機能

**関数**: `parseText()` → `buildPrompt()` → `api.php` 経由で Gemini API を呼び出し

#### 入力

- `#inputText` に入力された任意の文字列

#### 処理フロー

1. 入力テキストを `buildPrompt()` でプロンプトに変換
2. `api.php` に POST（Content-Type: `application/json`）
3. Gemini API のレスポンスから JSON を抽出（`{` ～ `}` の範囲）
4. JSON パース（失敗時は 3 段階のフォールバックロジックで再試行）
5. 成功 → `showResultPage()` を呼び出してページ 2 へ遷移

#### JSON パース フォールバック

| 段階 | 処理 |
|---|---|
| 1 | `JSON.parse()` 直接実行 |
| 2 | 制御文字・改行をエスケープして再実行 |
| 3 | 正規表現で各フィールドを個別抽出 |

#### 出力（AI が返す JSON 構造）

```json
{
  "title": "LAXUZ N2S1 北区 山田様",
  "startJST": "2026/05/10 10:00",
  "endJST": "2026/05/10 12:00",
  "location": "大阪市北区梅田1-1-1",
  "detail": "元のテキスト全文（改行は \\n エスケープ済み）"
}
```

---

### 3.2 タイトル生成ルール（プロンプト仕様）

Gemini に指示する生成ルール。`buildPrompt()` 関数に埋め込まれている。

#### 会社名の判定

| 入力テキストに含まれる文字列 | 出力される会社名 |
|---|---|
| 「おまかせ！」 | ユアマ公募 |
| 「高評価プロ」 | ユアマ公募 |
| 「ラクサス 050-6875-5609」 | LAXUZ |
| 「家族の健康の為に！」 | ユアマ |
| いずれも含まない | （空文字） |

#### 作業内容コード

複数の作業がある場合はコードを連結する（例: 通常2台+室外機1台 → `N2S1`）。

| コード | 作業内容 |
|---|---|
| N | 通常タイプ エアコン |
| R | お掃除機能付きエアコン |
| S | 室外機洗浄 |
| K | 消臭抗菌コート |
| E | エコ洗剤 |
| D | ドレンホース洗浄 |
| A | A セット |
| B | B セット |
| i | 防虫キャップ |
| T | ベランダ清掃 |
| W | 縦型洗濯機 |
| EL | レンジフード |
| V | 換気扇 |
| H | 空気清浄機 |

#### 訪問地の整形ルール

- 大阪市〇〇区 → **〇〇区のみ**出力
- その他市区町村 → **〇〇市のみ**出力

#### 氏名の整形ルール

- 「氏名：」または「お名前：」から**姓のみ**取り出し「〇〇様」とする

#### タイトルの結合形式

```
{会社名} {作業コード} {訪問地} {氏名}
```

例: `LAXUZ N2S1 北区 山田様`

---

### 3.3 日時変換機能

**関数**: `jstToUtc(jstStr)`

Google カレンダー URL に使用するため、JST → UTC に変換する。

#### 入力フォーマット

`YYYY/MM/DD HH:mm` または `YYYY-MM-DD HH:mm`（どちらも受け付ける）

#### 変換ロジック

```
UTC = JST - 9時間
```

#### 出力フォーマット（Google カレンダー URL 用）

`YYYYMMDDTHHmmssZ`（例: `20260510T010000Z`）

#### バリデーション

- フォーマット不一致の場合 `''` を返す
- `buildGcalUrl()` が空文字を受け取ると `alert()` でエラー表示して処理を中断

---

### 3.4 Google カレンダー URL 生成

**関数**: `buildGcalUrl()`

#### URL 形式

```
https://calendar.google.com/calendar/render?action=TEMPLATE
  &text={タイトル}
  &dates={startUTC}/{endUTC}
  &location={場所}
  &details={詳細}
```

- パラメータは `URLSearchParams` でエンコードする
- 生成した URL を `window.open(url, '_blank')` で開く

## 4. API 仕様（`api.php`）

### 4.1 エンドポイント

`POST /api.php`

### 4.2 リクエスト

| 項目 | 値 |
|---|---|
| Content-Type | `application/json` |
| メソッド | POST のみ（それ以外は 405） |
| ボディ | Gemini API 形式の JSON（フロントがそのまま転送） |

```json
{
  "contents": [
    { "parts": [{ "text": "プロンプト文字列" }] }
  ],
  "generationConfig": {
    "temperature": 0.1,
    "maxOutputTokens": 1024
  }
}
```

### 4.3 レスポンス

Gemini API のレスポンスをそのまま返す。

### 4.4 モデルフォールバック

| 優先順位 | モデル名 |
|---|---|
| 1 | `gemini-2.5-flash` |
| 2 | `gemini-2.0-flash` |
| 3 | `gemini-2.0-flash-lite` |

フォールバック条件: エラーメッセージに `high demand`・`quota`・`RESOURCE_EXHAUSTED` を含む場合。

### 4.5 CORS 設定

```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: POST
Access-Control-Allow-Headers: Content-Type
```

## 7. エラーハンドリング

| エラー状況 | 表示方法 |
|---|---|
| テキスト未入力 | `#errorArea` に `error-box` クラスで表示 |
| API レスポンスエラー | エラーメッセージを `error-box` で表示 |
| JSON 解析失敗 | フォールバック → 最終失敗時は `error-box` で表示 |
| 日時フォーマット不正 | `alert()` でポップアップ表示 |
| クリップボード読み取り失敗 | 静かに無視（エラー表示なし） |
| URL コピー失敗時の代替 | `execCommand('copy')` にフォールバック |

## 8. 非機能要件

| 項目 | 要件 |
|---|---|
| 認証 | なし（シングルユーザー前提） |
| 対応ブラウザ | モダンブラウザ（Chrome, Safari, Firefox 最新版） |
| 通信 | HTTPS 環境での利用を推奨 |
| オフライン | 非対応（API 通信が必須） |
| デプロイ | PHP が動くサーバーであれば何でも可 |
| 外部依存 | Google Fonts CDN、Gemini API のみ |
