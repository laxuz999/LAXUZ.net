# CLAUDE.md — カレンダー登録アプリ

このファイルは AI（Claude 等）がこのリポジトリを操作・改修する際の案内書です。
コードを触る前に必ず読んでください。

---

## プロジェクト概要

エアコン・家電クリーニング業者（LAXUZ / ユアマ 系）向けの **業務スケジュール登録 Web アプリ**。

作業依頼の通知テキストをクリップボードから貼り付けると、Gemini API が解析して
Google カレンダーへの登録リンクを生成する。ユーザーはリンクを踏むだけで予定が入る。

---

## ファイル構成

```
calendar/
├── CLAUDE.md          # 本ファイル（AI 向け案内書）
├── Spec.md            # 機能・画面・API 仕様書
├── config.php         # 定数定義（API キー等）
├── api.php            # Gemini API プロキシ（バックエンド）
└── calendar-app.html  # メインアプリ（フロントエンド、単一 HTML）
```

### 各ファイルの役割

| ファイル | 役割 | 変更頻度 |
|---|---|---|
| `config.php` | `GEMINI_API_KEY` の定義のみ | 低（初期設定のみ） |
| `api.php` | POST を受け Gemini API を呼ぶプロキシ。モデルフォールバックあり | 低 |
| `calendar-app.html` | UI・ロジックすべてが入った単一 HTML | 高 |

## 技術スタック

| 層 | 技術 |
|---|---|
| フロントエンド | バニラ HTML5 / CSS3 / JavaScript (ES2020+) |
| バックエンド | PHP 8.x（プロキシのみ） |
| AI 解析 | Google Gemini API（`gemini-2.5-flash` 優先、混雑時フォールバック） |
| カレンダー連携 | Google カレンダー URL スキーム（`calendar.google.com/calendar/render?action=TEMPLATE`） |
| フォント | Noto Sans JP / IBM Plex Mono（Google Fonts CDN） |

> **フレームワーク・ビルドツールは使用しない。** npm, webpack, vite 等は導入しないこと。

## コーディング規約

### 全般
- **文字コード**: UTF-8
- **インデント**: スペース 2 文字
- **言語**: 日本語コメント推奨（UI 文字列はすべて日本語）
- **外部ライブラリの追加は原則禁止**（CDN 経由も含む）

### HTML / CSS
- CSS 変数（`:root` の `--xxx`）でテーマカラーを管理する。ハードコードしない
- ダークテーマ固定。ライトモード切替は不要
- モバイルファースト設計。`max-width` は設定しない

### JavaScript
- `async/await` を使う（コールバック・`.then()` チェーンは使わない）
- DOM 操作は `getElementById` / `querySelector` を使う（jQuery 禁止）
- エラーは `try/catch` で囲み、`#errorArea` に `error-box` クラスで表示する
- `console.log` はデバッグ用のため、本番コミット前に削除する

### PHP
- `config.php` は `require_once` で読み込む
- CORS ヘッダーは `api.php` の先頭で設定済み。変更しないこと
- エラー出力は JSON 形式（`{'error': 'message'}`）で返す

---

## Gemini API の呼び出しルール

- **モデルフォールバック順**: `gemini-2.5-flash` → `gemini-2.0-flash` → `gemini-2.0-flash-lite`
- フォールバック条件: `high demand` / `quota` / `RESOURCE_EXHAUSTED` を含むエラー
- `temperature: 0.1`、`maxOutputTokens: 4096` 固定
- プロンプトは `calendar-app.html` の `buildPrompt()` 関数で組み立てる

---

## プロンプトの改修ガイド

`buildPrompt()` 内のプロンプトを編集する際の注意事項。

1. **出力形式は JSON のみ**（コードブロック・前置き禁止）を必ず明記する
2. `detail` フィールドの改行は `\n` エスケープを要求すること
3. 会社名・作業内容コードの対応表が変わった場合はこのセクションも更新する
4. プロンプト変更後は必ず実際のテキストで動作確認すること

### 現在の会社名ルール

|---|---|

| 「家族の健康の為に！」 | ユアマ |

## よくある改修パターン

### 1. 会社名・判定ルールの追加
`buildPrompt()` の `## 会社名` セクションに行を追加するだけ。

### 2. 作業内容コードの追加
`buildPrompt()` の `## 作業内容コード` セクションに `X=新しい作業` の形式で追記する。

### 3. UI テキストの変更
`calendar-app.html` 内で直接検索して変更する。

### 4. Gemini モデルの変更
`api.php` の `$models` 配列の先頭を変更する（フォールバック順に並べる）。

## セキュリティ注意事項

- `GEMINI_API_KEY` は `config.php` にのみ記載し、HTML や JS には絶対に書かない
- `config.php` は `.gitignore` に追加して Git 管理外にすること
- `api.php` は POST 以外のリクエストを 405 で拒否している（変更しないこと）
- クリップボードアクセスは `try/catch` で囲み、失敗してもアプリが動くようにする

## テスト方法

ビルドツールがないため、ブラウザで直接動作確認する。

1. PHP サーバーを起動: `php -S localhost:8080`
2. `http://localhost:8080/calendar-app.html` を開く
3. 以下のサンプルテキストで動作を確認する

```
【ラクサス 050-6875-5609】
作業日時：2026/05/10 10:00〜12:00
氏名：山田 太郎
住所：大阪市北区梅田1-1-1
エアコン通常タイプ 2台 室外機洗浄 1台
```

期待される解析結果:
- タイトル: `LAXUZ N2S1 北区 山田様`
- 開始: `2026/05/10 10:00`
- 終了: `2026/05/10 12:00`

## やってはいけないこと

- `config.php` の API キーをハードコードしたまま Git にコミットする
- npm / pip などのパッケージマネージャを導入する
- 外部 CSS フレームワーク（Bootstrap、Tailwind 等）を導入する
- 認証機能を追加する（シングルユーザー前提のため不要）
- フロントエンドを React / Vue / Svelte 等に書き直す（バニラ JS を維持する）
